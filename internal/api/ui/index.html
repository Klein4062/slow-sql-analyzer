PostgreSQL 计划分析器

输入 SQL 或执行计划，诊断是否最优并给出可执行建议 分析规则说明 →
实时（连库 EXPLAIN）
离线（粘贴 EXPLAIN JSON）

占位符 {dsn}/{sql} 或环境变量 $SSA_DSN/$SSA_SQL/$SSA_TIMEOUT。

无需数据库即可体验
如何获取离线计划？ 
# 在有 PG 的机器上导出 EXPLAIN JSON（-t -A 去掉表头/对齐，只剩纯 JSON）：
psql -d mydb -t -A -c "EXPLAIN (ANALYZE, BUFFERS, FORMAT JSON) SELECT * FROM orders WHERE status='pending'"

# 或存成文件再粘贴到上面文本框：
psql -d mydb -t -A -c "EXPLAIN (ANALYZE, BUFFERS, FORMAT JSON) <你的 SQL>" > plan.json

把输出（顶层的 JSON 数组）整段粘贴到上方文本框，点「分析」即可。

实时模式在只读事务内执行并设置 statement_timeout；写语句需勾选「允许写语句」。

在左侧输入查询或粘贴计划，点击「分析」查看结果。
